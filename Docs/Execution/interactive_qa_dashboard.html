🔑 Hướng dẫn sử dụng Token trên Swagger UI:

Hầu hết các API chỉnh sửa (POST, PATCH, DELETE) đều được cấu hình Auth Guard chặt chẽ để chống lỗi bảo mật leo thang đặc quyền:

  1. Khi chạy API Đăng nhập (Login), copy toàn bộ giá trị trong trường access_token của JSON trả về.
  2. Cuộn lên trên cùng của trang Swagger UI, nhấn nút Authorize màu xanh.
  3. Dán Token vừa copy vào ô dữ liệu và nhấn Authorize để áp dụng token cho tất cả các thao tác tiếp theo.

1. Đăng ký tài khoản, OTP & Xác thực

Kiểm tra luồng tạo tài khoản ở trạng thái chưa active và kích hoạt OTP trên môi trường cloud.

Bước 1.1 Đăng ký tài khoản Volunteer mới
🖥️ Trên Frontend đã Deploy
  1. Truy cập Frontend Dev URL.
  2. Nhấn Đăng ký (Register).
  3. Nhập Email mới chưa từng đăng ký, Số điện thoại và Họ tên.
  4. Nhấn nút Đăng ký → Màn hình sẽ chuyển sang trang nhập mã OTP.
🛠️ Trên Swagger UI
POST /api/v1/auth/register

Mở Swagger UI → auth/register → Nhấn Try it out → Nhập body và chạy:

{
  "email": "volunteer_cloud_test@example.com",
  "password": "Password123!",
  "phone_number": "+84912345678",
  "full_name": "Nguyễn Văn Test Cloud"
}
Xác nhận tài khoản đăng ký thành công (trả về trạng thái pending_otp).
Bước 1.2 Xác thực mã OTP
🖥️ Giao diện Frontend
  1. Nhập mã OTP mặc định để debug là: 123456.
  2. Nhấn Xác nhận.
  3. Xác nhận thông báo kích hoạt thành công xuất hiện và chuyển hướng tới trang Đăng nhập.
🛠️ Trên Swagger UI
POST /api/v1/auth/verify-otp

Mở API auth/verify-otp và gửi mã kích hoạt:

{
  "email": "volunteer_cloud_test@example.com",
  "otp_code": "123456"
}
Xác nhận tài khoản chuyển sang trạng thái active. Đăng nhập với mật khẩu đã tạo thành công.

2. Luồng Nâng Cấp Quyền Organizer (Ban Tổ Chức)

Kiểm thử luồng gửi yêu cầu nâng quyền của Volunteer và sự phê duyệt bảo mật của Admin.

Bước 2.1 Volunteer gửi đơn xin nâng quyền
🖥️ Giao diện Frontend
  1. Đăng nhập tài khoản Volunteer vừa tạo trên Web.
  2. Vào mục Cá nhân → Nhấn Đăng ký làm Ban tổ chức (Organizer).
  3. Điền lý do và kinh nghiệm → Click Gửi đơn.
  4. Màn hình sẽ hiển thị trạng thái: Đang chờ duyệt (Pending).
🛠️ Trên Swagger UI
POST /api/v1/organizer-requests/request-upgrade

1. Gọi API auth/login để lấy access token của Volunteer và nhấn Authorize trên Swagger.

2. Gọi API gửi đơn nâng cấp với cấu trúc trường phẳng mới đồng bộ:

{
  "reason": "Tổ chức chiến dịch phân loại rác thải",
  "experience": "Hoạt động xã hội 2 năm",
  "contact_phone": "+84912345678"
}
Xác nhận hồ sơ gửi lên thành công không bị lỗi Validate Pydantic 422.
Bước 2.2 Admin phê duyệt đơn nâng cấp
🖥️ Giao diện Frontend
  1. Đăng xuất tài khoản Volunteer.
  2. Đăng nhập bằng tài khoản Admin hệ thống: admin@volunteerconnect.org / AdminSecurePassword123!.
  3. Vào mục Admin Dashboard → danh sách đơn xin nâng quyền.
  4. Nhấn nút Duyệt (Approve).
🛠️ Trên Swagger UI
PATCH /api/v1/admin/requests/{request_id}/approve

1. Lấy token Admin bằng API login → Nhấn Authorize trên Swagger.

2. Lấy ID đơn vừa gửi bằng API: GET /api/v1/admin/organizer-requests.

3. Gửi lệnh duyệt (sử dụng phương thức PATCH chuẩn bảo mật mới):

{
  "status": "approved",
  "reason": "Yêu cầu hợp lệ"
}
Xác nhận tài khoản Volunteer được chuyển đổi vai trò thành organizer thành công.

3. Vòng Đời Hoạt Động (Draft $\rightarrow$ Submit $\rightarrow$ Approve $\rightarrow$ Open)

Kiểm thử luồng đăng hoạt động mới và quy trình phê duyệt hoạt động của Ban quản trị hệ thống.

Bước 3.1 Organizer tạo hoạt động & Gửi duyệt
🖥️ Giao diện Frontend
  1. Đăng nhập tài khoản Organizer mới được duyệt ở Kịch bản 2.
  2. Vào Organizer Dashboard → Click Tạo hoạt động mới.
  3. Đặt giới hạn số lượng TNV (limit_volunteers) là 2 người → Lưu bản nháp (Draft).
  4. Nhấn nút Gửi kiểm duyệt (Submit).
🛠️ Trên Swagger UI
POST /api/v1/activities & /submit

1. Gửi lệnh tạo hoạt động dưới Token của Organizer:

{
  "title": "Chương trình Dọn rác Vũng Tàu",
  "description": "Hoạt động bảo vệ môi trường biển cuối tuần sạch đẹp xanh mát",
  "categories": ["Môi trường"],
  "location": {
    "province": "Bà Rịa - Vũng Tàu",
    "district": "Vũng Tàu",
    "address_detail": "Bãi Sau"
  },
  "start_date": "2026-09-01T08:00:00Z",
  "end_date": "2026-09-02T17:00:00Z",
  "limit_volunteers": 2,
  "requirements": "Không ngại nắng nóng"
}

2. Gửi duyệt hoạt động bằng API: POST /api/v1/activities/{activity_id}/submit.

Xác nhận hoạt động được tạo ở trạng thái draft and gửi duyệt thành công sang trạng thái pending_review.
Bước 3.2 Admin duyệt hoạt động
🖥️ Giao diện Frontend
  1. Đăng nhập tài khoản Admin.
  2. Mở Admin Dashboard → mục duyệt hoạt động.
  3. Nhấn nút Duyệt (Approve).
🛠️ Trên Swagger UI
PATCH /api/v1/admin/activities/{activity_id}/approve

Admin gửi yêu cầu phê duyệt hoạt động:

{
  "is_approved": true,
  "reason": "Hoạt động thiết thực"
}
Xác nhận hoạt động chuyển sang trạng thái open chuẩn xác (đã được sửa lỗi PUBLISHED status cũ) và xuất hiện trên trang khám phá của TNV.

4. Đăng Ký Đơn & Tự Động Quản Lý Sức Chứa (Capacity)

Kiểm thử cơ chế tự động chuyển đổi thông minh giữa hai trạng thái Open (Mở tuyển) và Full (Hết chỗ) khi có biến động số lượng đăng ký.

Bước 4.1 Tuyển đủ số lượng TNV (Chuyển sang FULL)
🖥️ Giao diện Frontend
  1. Dùng 2 tài khoản Volunteer khác đăng ký tham gia sự kiện.
  2. Organizer vào trang quản lý hoạt động → Click duyệt (Approve) đơn đăng ký của cả 2 Volunteer.
  3. Quay lại trang khám phá hoạt động.
  4. Xác nhận trạng thái hoạt động tự động chuyển sang Hết chỗ (Full).
🛠️ Trên Swagger UI
PATCH /api/v1/registrations/{registration_id}/approve

Duyệt đơn đăng ký của TNV bằng API PATCH chuẩn mới (trước đây Frontend gọi sai bằng POST):

curl -X PATCH https://volunteer-connect-backend-dev-iz4fa4z53q-as.a.run.app/api/v1/registrations/<REGISTRATION_ID>/approve \
-H "Authorization: Bearer <ORGANIZER_TOKEN>"
Xác nhận khi số lượng đơn được duyệt chạm mốc limit (2), status chuyển sang full. Đơn ở trạng thái pending (chưa duyệt) không được chiếm chỗ.
Bước 4.2 Hủy đăng ký (Chuyển ngược về OPEN)
🖥️ Giao diện Frontend
  1. Đăng nhập bằng 1 trong các tài khoản Volunteer đã được duyệt tham gia ở trên.
  2. Vào trang **Hoạt động của tôi** → Chọn sự kiện → Click **Hủy đăng ký** (Cancel).
  3. Quay lại trang khám phá chính, xác nhận hoạt động tự động quay lại trạng thái **Mở tuyển (Open)**.
🛠️ Trên Swagger UI
POST /api/v1/registrations/{registration_id}/cancel

Volunteer gửi lệnh hủy đơn bằng API:

curl -X POST https://volunteer-connect-backend-dev-iz4fa4z53q-as.a.run.app/api/v1/registrations/<REGISTRATION_ID>/cancel \
-H "Authorization: Bearer <VOLUNTEER_TOKEN>"
Xác nhận khi số lượng thành viên thực tế giảm xuống dưới mốc limit, hoạt động mở lại trạng thái open ngay lập tức.

5. Điểm Danh Tình Nguyện Viên (Attendance)

Kiểm thử tính năng điểm danh và cập nhật điểm tích lũy hoạt động tham gia của Volunteer.

Điểm danh Ghi nhận tham dự và tích điểm
🖥️ Giao diện Frontend
  1. Đăng nhập tài khoản Organizer sở hữu hoạt động.
  2. Vào trang quản lý hoạt động → Click xem danh sách Tình nguyện viên tham gia.
  3. Tích chọn trạng thái **Có mặt (Completed)** cho các TNV có mặt thực tế.
  4. Nhấn nút **Lưu điểm danh**.
🛠️ Trên Swagger UI
PATCH /api/v1/registrations/{registration_id}/attendance

Điểm danh an toàn bằng API PATCH đồng bộ mới (Frontend gọi sai bằng POST check-in cũ):

{
  "status": "completed"
}
Xác nhận thông tin điểm danh được lưu trữ, kiểm tra profile Volunteer có điểm joined_activity_count tăng lên 1 điểm.

6. Bảng Tin Cộng Đồng (Feed & Posts)

Kiểm thử tính năng tương tác đăng bài viết lên bảng tin cộng đồng, thích bài viết (like) và bảo mật quyền xóa bài viết.

Bảng tin Tương tác bài đăng trên Feed
🖥️ Giao diện Frontend
  1. Đăng nhập tài khoản bất kỳ → Vào mục **Bảng tin** (Feed).
  2. Nhập nội dung chia sẻ, tải ảnh minh họa → nhấn **Đăng bài**.
  3. Nhấn nút **Thích (Like)** → xác nhận số lượng thích tăng lên.
  4. Nhấn **Xóa** bài đăng của chính mình → bài đăng biến mất khỏi bảng tin.
🛠️ Trên Swagger UI
POST /api/v1/posts/ & PATCH /like & DELETE

Tương tác bảng tin Feed an toàn qua tiền tố prefix `/api/v1/posts` mới sửa đổi:

{
  "title": "Chương trình dọn rác xanh",
  "content": "Một ngày cuối tuần dọn dẹp bãi biển đầy ý nghĩa tại Vũng Tàu!",
  "images": ["https://picsum.photos/200"],
  "hashtags": ["vungtau", "moitruong"]
}
Xác nhận luồng Feed hoạt động bình thường, không bị lỗi prefix đường dẫn và được bảo vệ quyền sở hữu an toàn (chỉ tác giả hoặc admin mới xóa được bài).
